# 管理层业务数据模块说明

> 本文档描述财务/管理层用户可查看的五大业务数据模块的功能和数据字段

---

## 一、签约模块

**页面路径：** `pages/stats/farmers/index`

### 1.1 功能概述
管理层查看农户签约的总体情况和明细数据，支持按昨日新增和全年累计两个维度查看。

### 1.2 核心数据字段

#### 汇总统计
| 字段 | 说明 | 示例 |
|------|------|------|
| 签约农户总数 | 累计签约的农户数量 | 7000户 |
| 签约总面积 | 所有签约农户的种植面积总和 | 2.1万亩 |
| 收取押金总额 | 所有农户缴纳的押金总和 | 350万 |

#### 农户等级分布
| 字段 | 说明 |
|------|------|
| 金牌农户 | 数量及占比 |
| 银牌农户 | 数量及占比 |
| 铜牌农户 | 数量及占比 |

#### 农户列表明细
| 字段 | 说明 |
|------|------|
| 农户姓名 | - |
| 手机号 | - |
| 等级 | 金/银/铜 |
| 种植面积 | 亩 |
| 押金 | 元 |
| 负责人 | 业务员姓名 |
| 签约日期 | - |

### 1.3 筛选功能
- 时间维度：昨日新增 / 全年累计
- 搜索：按农户姓名或手机号
- 等级筛选：金牌 / 银牌 / 铜牌
- 负责人筛选：选择业务员

## 二、发苗模块

**页面路径：** `pages/stats/seeds/index`

### 2.1 功能概述
管理层查看种苗发放的统计数据和明细记录，支持按昨日和全年度累计查看。

### 2.2 核心数据字段

#### 汇总统计
| 字段 | 说明 | 示例 |
|------|------|------|
| 发放总量 | 种苗发放的总重量 | 580吨 |
| 总金额 | 发放种苗的总价值 | 696万 |
| 发放农户 | 领取种苗的农户数 | 5810户 |
| 已收款 | 农户已支付的金额 | 522万 |
| 欠款 | 农户未支付的金额 | 174万 |

#### 发放进度（全年）
| 字段 | 说明 |
|------|------|
| 进度百分比 | 已发放农户 / 签约总农户 |
| 已发放农户数 | - |
| 待发放农户数 | - |

#### 发放趋势图
- 按日统计（本周7天）
- 按周统计（本月4周）
- 按月统计（本年12月）

#### 发放记录列表
| 字段 | 说明 |
|------|------|
| 农户姓名 | - |
| 手机号 | - |
| 品种 | 甜叶菊等 |
| 数量 | kg |
| 金额 | 元 |
| 收款状态 | 已收款/部分收款/未收款 |
| 发放日期 | - |
| 发放人 | 业务员姓名 |

### 2.3 筛选功能
- 时间维度：昨日发放 / 全年度累计
- 搜索：按农户姓名或手机号
- 负责人筛选
- 收款状态筛选

## 四、收苗模块（仓库）

**页面路径：** `pages/stats/acquisition/index`  
**详情页路径：** `pages/stats/warehouse-detail/index`

### 4.1 功能概述
管理层查看收苗（农产品收购）的统计数据，按仓库分类管理，支持查看各仓库详情。

### 4.2 核心数据字段

#### 汇总统计
| 字段 | 说明 | 示例 |
|------|------|------|
| 收购总重量 | 公斤（甲方要求） | 1010000kg |
| 总金额 | 收购支付的总金额 | 707万 |
| 单价 | 总金额 / 总重量 | 7元/kg |
| 已卖叶农户 | 交售过农产品的农户数（仅累计显示） | 5680户 |

#### 仓库分类统计（10个仓库）

**仓库容量配置：**
| 类型 | 仓库 | 容量 | 用途 |
|------|------|------|------|
| 大型仓库 | 一号、二号、三号 | 200吨 | 长期存储和调度汇总 |
| 中型仓库 | 四号、五号、六号 | 150吨 | 常规收苗点 |
| 小型仓库 | 七号~十号 | 100吨 | 临时收苗点，满仓后调度到大仓库 |

**每个仓库显示字段：**
| 字段 | 说明 |
|------|------|
| 仓库名称 | 一号~十号仓库 |
| 仓库类型 | 大/中/小 |
| 当日/累计收购重量 | kg |
| 金额 | 元 |
| 单价 | 元/kg |
| 当前库存 | kg |
| 总容量 | kg |
| 使用率 | 百分比 |
| 容量状态 | 正常(<80%) / 预警(80%-94%) / 满仓(≥95%) |

### 4.3 仓库详情页
点击仓库卡片进入详情页，显示：

#### 仓库概览
- 仓库名称、类型
- 容量使用情况（当前库存/总容量/剩余容量/使用率）
- 收购总重量（kg）、总金额、单价、交苗农户数

#### 收苗明细列表
| 字段 | 说明 |
|------|------|
| 农户姓名 | - |
| 手机号 | - |
| 重量 | kg |
| 单价 | 元/kg |
| 金额 | 元 |
| 日期时间 | - |
| 录入人 | 仓库管理员 |

### 4.4 筛选功能
- 时间维度：今日数据 / 累计数据
- 搜索：按农户姓名或手机号

## 五、结算模块

**页面路径：** `pages/finance/index/index`  
**详情页路径：** `pages/finance/detail/index`

### 5.1 功能概述
管理层查看农户结算情况，包括待审核、待支付、已完成等状态的管理。

### 5.2 核心数据字段

#### 汇总统计
| 字段 | 说明 | 示例 |
|------|------|------|
| 应付总额 | 所有农户的应付金额总和 | 4200万 |
| 已付总额 | 已经支付给农户的金额 | 3150万 |
| 待付总额 | 还未支付的金额 | 1050万 |
| 预付总额 | 预付给农户的金额 | 280万 |

#### 结算状态分类
| 状态 | 说明 |
|------|------|
| 全部 | 所有结算记录 |
| 待审核 | 需要审核的结算申请 |
| 待支付 | 审核通过，等待支付 |
| 已完成 | 已完成支付 |

#### 结算列表字段
| 字段 | 说明 |
|------|------|
| 农户姓名 | - |
| 手机号 | - |
| 结算状态 | 待审核/待支付/已完成 |
| 应付金额 | 最终应付 |
| 已付金额 | 已支付 |
| 待付金额 | 剩余未付 |
| 收购金额 | 农产品收购总额 |
| 扣款金额 | 种苗款、农资款等扣除 |
| 支付进度 | 已付/应付 百分比 |

### 5.3 结算详情页
点击结算记录进入详情，显示：
- 农户基本信息
- 结算金额明细（收购款、扣款项、应付款）
- 支付记录历史
- 操作按钮（审核、支付等）

### 5.4 筛选功能
- 状态筛选：全部 / 待审核 / 待支付 / 已完成
- 搜索：按农户姓名或手机号
- 分页加载：支持7000+农户的大数据量
